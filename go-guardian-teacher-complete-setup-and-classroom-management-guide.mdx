## Step 1: Install the Chrome Extension

The GoGuardian Chrome extension is the foundation of classroom management. IT administrators deploy it through Google Admin Console.

### For IT Administrators

Navigate to Google Admin Console and follow these steps:

```text
Google Admin Console → Devices → Chrome → Apps & Extensions → Users & Browsers
```

Search for **GoGuardian** in the Chrome Web Store and configure the force-install policy:

```json
{
  "ExtensionSettings": {
    "goguardian-extension-id": {
      "installation_mode": "force_installed",
      "update_url": "https://clients2.google.com/service/update2/crx",
      "toolbar_pin": "force_pinned"
    }
  }
}
```

### For Teachers

Once the extension is deployed, verify it appears in a student's Chrome browser:

1. Open Chrome on any student device
2. Click the **Extensions** icon (puzzle piece) in the toolbar
3. Confirm **GoGuardian** is listed and enabled
4. If not visible, contact your IT administrator

<Tip>
  You can verify the extension is working by starting a test session with a single device before rolling out to the entire class.
</Tip>

## Step 2: Create Your First Classroom Session

### Starting a Session

```text
GoGuardian Dashboard → Teacher → Classes → Select Class → Start Session
```

GoGuardian automatically syncs with **Google Classroom** rosters. If your classes are already set up in Google Classroom, you'll see them listed immediately.

### Session Configuration Options

Here's an example of how GoGuardian sessions can be configured programmatically (for IT admins integrating via API):

```python
# GoGuardian API session configuration
session_config = {
    "class_id": "math-101-period-3",
    "teacher_id": "teacher@school.edu",
    "duration_minutes": 50,
    "auto_end": True,
    "scenes": ["default_browsing"],
    "features": {
        "screen_monitoring": True,
        "chat_enabled": True,
        "announcements": True,
        "tab_control": True,
        "lock_screen": False
    }
}

# Start the session
response = goguardian.sessions.create(session_config)
print(f"Session started: {response.session_id}")
```

<Info>
  Sessions can also be **scheduled** to start and stop automatically. This is ideal for teachers with consistent daily schedules — set it once and it runs all semester.
</Info>

## Step 3: Configure Scenes for Content Filtering

**Scenes** are browsing rule sets that control which websites students can access during a session. Think of them as customizable filters you can switch between instantly.

### Scene Types

<CardGroup cols={2}>
  <Card title="Open Browsing" icon="globe">
    Students can access any website not blocked by the district-level filter. Best for research projects and free exploration time.
  </Card>

  <Card title="Focused Browsing" icon="bullseye">
    Only pre-approved websites are accessible. Ideal for test-taking, specific assignments, or guided activities.
  </Card>

  <Card title="Blocked List" icon="ban">
    All sites are allowed except those you specifically block. Good for removing common distractions like social media and games.
  </Card>

  <Card title="Custom Scene" icon="sliders">
    Mix of allowed and blocked sites tailored to a specific lesson or activity.
  </Card>
</CardGroup>

### Creating a Custom Scene

```javascript
// Creating a scene via the GoGuardian API
const scene = {
  name: "Math Research Project",
  type: "FOCUSED",
  allowedDomains: [
    "khanacademy.org",
    "desmos.com",
    "wolframalpha.com",
    "mathway.com",
    "docs.google.com"
  ],
  blockedDomains: [
    "youtube.com",
    "tiktok.com",
    "instagram.com",
    "reddit.com"
  ],
  settings: {
    allowGoogleSearch: true,
    searchSafeMode: "strict",
    allowGoogleDocs: true
  }
};

await goguardian.scenes.create(classId, scene);
```

### How Scenes Interact with District Filters

![GoGuardian Filter Hierarchy](https://www.goguardian.com/hubfs/admin-filtering.png)

District-level filters always take priority. Teacher Scenes operate within the boundaries set by the IT administrator. If a website is blocked at the district level, a teacher Scene cannot override it — but teachers can use the **Temporary Unlock** feature to request access for their session.

## Step 4: Set Up Student Groups

Student Groups allow you to apply different Scenes to different students simultaneously — perfect for differentiated instruction.

### Use Cases for Student Groups

<Accordion title="Test-Taking Scenario">
  **Situation**: Half the class is retaking a quiz while others work on a new assignment.

  - **Group A (Test Retake)**: Apply a locked-down Scene that only allows the testing platform
  - **Group B (Assignment)**: Apply an open browsing Scene with access to research sites

  Teachers can drag and drop students between groups in real-time as they finish their quiz.
</Accordion>

<Accordion title="Differentiated Learning Levels">
  **Situation**: Students at different reading levels working on different materials.

  - **Group 1 (Advanced)**: Access to primary source documents and academic journals
  - **Group 2 (On-Level)**: Access to grade-level articles and educational videos
  - **Group 3 (Support)**: Access to scaffolded materials and interactive learning tools

  Each group sees only the resources appropriate for their level.
</Accordion>

<Accordion title="Station Rotation Model">
  **Situation**: Students rotate through learning stations every 15 minutes.

  - **Station 1 (Direct Instruction)**: Screens locked to teacher presentation
  - **Station 2 (Independent Practice)**: Access to math practice platform only
  - **Station 3 (Research)**: Open browsing with content filter
  - **Station 4 (Assessment)**: Locked to quiz platform

  As students rotate, the teacher moves them to the appropriate group with drag-and-drop.
</Accordion>

## Step 5: Real-Time Monitoring and Commands

### Screen View

GoGuardian provides a live thumbnail view of every student's screen. The view refreshes faster than competing products, giving you an accurate picture of student activity.

### Teacher Commands Reference

| Command | Action | Keyboard Shortcut |
| --- | --- | --- |
| **Open Tab** | Opens a specific URL on all student devices | `Ctrl + Shift + O` |
| **Close Tab** | Closes a specific tab on all student devices | `Ctrl + Shift + C` |
| **Lock Screen** | Locks all student screens with a custom message | `Ctrl + Shift + L` |
| **Send Announcement** | Displays a pop-up message on all screens | `Ctrl + Shift + A` |
| **Focus Tab** | Forces all students to a single open tab | `Ctrl + Shift + F` |
| **Share Screen** | Shares your screen to all student devices | `Ctrl + Shift + S` |

### Private Chat

The two-way chat allows private communication with individual students:

```text
Teacher → Click student thumbnail → Chat icon → Type message → Send
```

<Note>
  Chat messages are logged and available in session reports. This provides documentation for parent-teacher conferences and administrative review.
</Note>

## Step 6: Schedule Recurring Sessions

For teachers with consistent schedules, **Scheduled Sessions** eliminate the need to manually start and stop sessions each day.

### Configuration Example

```yaml
# Schedule configuration for a typical teacher
schedule:
  class: "Math 101 - Period 3"
  timezone: "America/Los_Angeles"
  sessions:
    - days: [monday, tuesday, wednesday, thursday, friday]
      start: "09:00"
      end: "09:50"
      scene: "math-focused-browsing"
    - days: [wednesday]
      start: "14:00"
      end: "14:50"
      scene: "math-lab-open-browsing"
  exceptions:
    - date: "2026-12-20"
      reason: "Winter Break"
    - date: "2026-01-06"
      reason: "Professional Development Day"
```

## Step 7: Review Reports and Analytics

After each session, GoGuardian generates detailed reports on student activity.

### Available Report Types

<Steps>
  <Step title="Browsing History Report">
    Complete timeline of every website visited by each student during the session. Includes timestamps, duration, and tab switching patterns.
  </Step>
  <Step title="Off-Task Alert Summary">
    AI-powered analysis of student behavior. Flags students who spent significant time on non-educational sites or showed patterns of distraction.
  </Step>
  <Step title="Engagement Dashboard">
    Visual overview of class engagement levels throughout the session. Shows peaks and valleys in on-task behavior correlated with lesson activities.
  </Step>
  <Step title="Chat Transcript Log">
    Complete record of all teacher-student chat messages during the session. Useful for documentation and follow-up.
  </Step>
</Steps>

### Exporting Data via API

```bash
# Export session report (for administrators)
curl -X GET "https://api.goguardian.com/v2/reports/sessions/{session_id}" \
  -H "Authorization: Bearer ${API_TOKEN}" \
  -H "Accept: application/csv" \
  -o "session_report_2026-05-20.csv"
```

## Troubleshooting Common Issues

<Accordion title="Extension Not Appearing on Student Devices">
  **Cause**: Extension deployment may be pending in Google Admin Console.

  **Solution**:

  1. Verify the extension is force-installed in Google Admin Console
  2. Check the target OU (Organizational Unit) includes the student device
  3. Have the student restart Chrome or reboot the device
  4. Allow up to 24 hours for policy propagation

  ```bash
  # Check Chrome policy status on a student device
  # Navigate to: chrome://policy
  # Search for: GoGuardian
  # Status should show: OK
  ```
</Accordion>

<Accordion title="Students Not Appearing in Session">
  **Cause**: Roster sync issue between Google Classroom and GoGuardian.

  **Solution**:

  1. Confirm the student is enrolled in the Google Classroom class
  2. Check that the student is signed into Chrome with their school account
  3. Refresh the GoGuardian session
  4. Contact support: [**support@goguardian.com**](mailto:support@goguardian.com) or **1-855-GO-GUARD**
</Accordion>

<Accordion title="Scenes Not Applying Correctly">
  **Cause**: Conflicting district-level filter policies may override classroom Scenes.

  **Solution**:

  1. District filters always take priority over teacher Scenes
  2. If a site is blocked at the district level, teacher Scenes cannot unblock it
  3. Use the **Temporary Unlock** feature to request a site be allowed
  4. Coordinate with your IT administrator for permanent allow-list changes
</Accordion>

## Best Practices

<Note>
  These recommendations are based on feedback from over 1.9 million educators using GoGuardian Teacher across 15.3 million student devices.
</Note>

1. **Start every class with a Scene** — Set browsing expectations from the first minute
2. **Use Chat for quiet redirection** — Send a private message instead of calling out a student publicly
3. **Schedule sessions in advance** — Eliminate the daily overhead of starting and stopping sessions
4. **Review reports weekly** — Spot trends and address issues early
5. **Create reusable Scenes** — Build a library for different activity types (testing, research, free time)
6. **Leverage Student Groups** — Drag-and-drop makes differentiation easy without extra planning
